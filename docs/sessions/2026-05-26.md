### 3. LLM出力の途中切れバグ修正
**ファイル**: `backend/pii_detector.py`

**症状**: 4行のテキストを処理すると「はじめまして」（6文字）しか出力されない。

**原因**: LLMが前置き文（"以下が処理済みテキストです："など）でトークンを消費し、
`max_tokens=512` では本文を書き切れずに途中で切れていた。

**修正内容**:
- `max_tokens` を `max(512, len×2)` → `max(1024, len×4)` に拡大
- 出力が入力の50%未満の場合は正規表現結果にフォールバックするサニティチェックを追加

```python
if len(cleaned) < len(text) * 0.5:
    _log(f"[LLM WARN] 出力が短すぎます → 正規表現結果を使用")
    return text
```

### 4. 長文ファイルのハング問題修正（チャンク分割）
**ファイル**: `backend/pii_detector.py`

**症状**: 258行・9891文字のテキストファイルで処理が止まる。

**原因**: 9891文字 × 4 = 約40000トークンを一度のLLMリクエストで生成しようとしていた。

**修正内容**: テキストを1000文字単位（行境界で分割）に分けてLLMを複数回呼び出す仕組みを追加。

追加した関数:
| 関数 | 役割 |
|---|---|
| `_MAX_CHUNK_CHARS = 1000` | チャンクサイズ定数 |
| `_chunk_text(text)` | 行単位で1000文字以内に分割 |
| `_redact_chunk(text, sensitivity)` | 1チャンク分のLLM処理 |
| `_redact_with_llm(text, sensitivity)` | チャンク分割→各処理→結合のオーケストレーター |

処理中のログ例:
```
[LLM] テキスト長=9891字
[LLM] 10チャンクに分割して処理します
[LLM] チャンク 1/10 (長さ=987字)
[LLM] チャンク 2/10 (長さ=1000字)
...
```

タイムアウトが発生する場合は `_MAX_CHUNK_CHARS = 1000` を `500` に下げる。
