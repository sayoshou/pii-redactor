### 2. `.txt` アップロードで `unsupported file type` エラー（バグ修正）

**症状**: LM Studioを起動した状態で `.txt` ファイルをアップロードすると `unsupported file type` エラーになる。

**原因調査**:
- `context.md` の改善履歴（項目8）には「2026-05-26に `.txt` 対応を追加した」と記載されていた。
- しかし `git log -- pii_redactor/backend/app.py` を確認したところ、初期コミット以降 `app.py` への変更は一度もなく、`.txt` 分岐は実装されていなかった。
- おそらく過去のセッションで実装した内容がコミットされずに失われた（ドキュメントだけが先行して残った）ケース。

**修正内容**: `pii_redactor/backend/app.py` の `/api/redact` に `.txt` 分岐を追加。

```python
elif suffix in ["txt"]:
    try:
        text = content.decode("utf-8")
    except UnicodeDecodeError:
        text = content.decode("cp932", errors="replace")
    redacted = redact_pii(text)
    output_bytes = redacted.encode("utf-8")
    mimetype = "text/plain"
```

**動作確認**: サンプルテキスト（氏名・メール・電話番号を含む）をアップロードし、200 OKでメール・電話番号がマスクされることを確認。

**副次的に判明した既知の課題**: 上記確認で氏名（田中太郎）が `[NAME]` に置換されず本文から消失していた。LLM（Gemma）の名前検出精度が低いという既知の課題（Obsidianノートに記載済み）に該当する別問題であり、今回は対応していない。
